Installazione

  1. Con riferimento a compose-mysql-withvolume.yml si esegue 
    docker-compose -f compose-mysql-withvolume.yml up
    che installa:
    • il servizio db con immagine mysql settando nell'environment variabili relative al nome del db e alla password ed sponendo la porta 3306.;
    • il servizio adminer con immagine adminer con porta 8018/8080 per la gestione dei db;
    • il servizio mc con dipendenza da db e con immagine nextgenhealthcare/connect, settando nell'environment variabili relative al database. Inoltre definisce in volumes il mapping tra una directory del PC (C:\Progetti\natmaterial\mirth-connect-channel-examples\data\volumes\appdata) e una directory della VM (/opt/connect/appdata).
      Includere nel file .gitignore la directory workingok/data/volumes/appdata/temp
  2. aprire http://localhost:8080/ e click on Launch Mirth Connect Administrator che scarica un file webstart.jnlp. Click su questo file e si apre Mirth Connect Administrator Launcher che propone http://localhost:8080/webstart. Premere il pulasante Launch per veder comparire Mirth Connect Login. Immettere Username=admin e Password=admin per fare comparire una form da compilare.

  3. aprire la GUI di adminer eseguendo localhost:8018 e settando opportunamente i parametri: 
    Sistema: MySQL
Server: db
Utente: root
Password: root_password
Database: mirthdb
    Si consulta il db che è già popolato in quanto sosituisce derby. Si noti che la tabella CHANNEL e' vuota.
    La tabella PERSON contiene dati, esportando la quale si ottiene il file PERSON.sql.
    Per l'uso del db all'interno di un canale Mirth, si rammenti: 
     DRIVER: com.mysql.jdbc.Driver
 URL:    jdbc:mysql://db:3306/mirthdb
  4. carichiamo user-defined Java code secondo nextgen-connect-39-user-guide.pdf (pag 157): in Settings->Resources selezionare Add Resource con nome unibo e directory /opt/connect/appdata/unibolibs. Premendo refresh si popola la finestra Loaded Libraries con tutti i file contenti nella dir specificata.
    Now, restart the Mirth Connect server by clicking again on webstartmysql.jnlp
  5. carichiamo l'estensione FHIR Listner predendo il file fhir-3.9.0.b1285.zip da una directory su PC.
    Now, restart the Mirth Connect server by clicking again on webstartmysql.jnlp

    • /appdata/mirthdb: The embedded database (Do NOT delete if you specify Derby as your database).

    • /cli-lib: Libraries for the NextGen Connect Command Line Interface (if installed)
    • /client-lib: Libraries for the NextGen Connect Administrator
    • /conf: Configuration files
    • /custom-lib: Place your custom user libraries here to be used by the default library resource.
    • /docs: This document and a copy of the NextGen Connect license
    • /docs/javadocs: Generated javadocs for the installed version of NextGen Connect.
      These documents are also available when the server is running at http://[server address]:8080/javadocs/ (i.e. http://localhost:8080/javadocs/).
    • /extensions: Libraries and meta data for Plug-ins and Connectors
    • /logs: Default location for logs generated by NextGen Connect and its sub-components
    • /manager-lib: Libraries for the NextGen Connect Server Manager (if installed)
    • /public_html: Directory exposed by the embedded web server
    • /server-launcher-lib: Libraries in this directory will be loaded into the main NextGen Connect Server thread context
      classloader upon startup. This is required if you are using any custom log4j appender libraries.
    • /server-lib: NextGen Connect server libraries
    • /webapps: Directory exposed by the embedded web server to host webapps

insertFhirResource

Il connector FHIR_Listener_create.xml operation invoca l'operazione insertFhirResource definita in Insert Resource del code template FHIR DB Operations.

Questa operazione, predisposta per i db postgres e sqlserver, e' stata modificata come riportato a fianco in relazione all'uso di MySQL.

I dati ricevuti nel campo di indice 4 di params rappresentano un paziente i cui valori vengono inseriti nel db (con executeUpdate).

function insertFhirResource(type, id, versionId, lastUpdated, data, contentType, method, url) {
	var dbType = $cfg('fhirDBDatabaseType'); //fhirDBDatabaseType definito in Configuration Map
	if (dbType == 'MySQL') {
		var params = [type, id, versionId, lastUpdated, data, contentType, method, url]; //come postgres
 		var dbConn;
		try { 
			var patientdata = params[4];
			logger.info("%%% patientdata=  " + patientdata);
			dbConn = DatabaseConnectionFactory.createDatabaseConnection(
			    'com.mysql.jdbc.Driver','jdbc:mysql://db:3306/mirthdb','root','root_password');
			
			var patientxml = new XML( patientdata );	
			
			//Accesso ai campi del paziente
			var UserName  = patientxml.name.use.@value;
			var LastName  = patientxml.name.family.@value;
			var FirstName = patientxml.name.given.@value;
			logger.info("%%% patientdata xml LastName=  " + LastName);
			
			var result = dbConn.executeUpdate(
				"INSERT INTO PERSON (USERNAME, FIRSTNAME, LASTNAME, LOGGED_IN)"+
				" VALUES ('"+ UserName +"','"+ FirstName +"','"+LastName +"', 0)");			
		}catch (e) { throw 'MySQL db ERROR: ' + e; }
			finally { if (dbConn) {  dbConn.close(); } 
		}		
 	} else
	if (dbType == 'postgres') {
	var params = [type, id, versionId, lastUpdated, data, contentType, method, url];
	var result = executeUpdateAndGetGeneratedKeys(
	"INSERT INTO resource(name,id,version,last_modified,data,mimetype,request_method,request_url)"+ 
	"VALUES (?, ?, ?, ?, ?::XML, ?, ?, ?)", params);
	result.next();
	return result.getInt(1);
	} else if (dbType == 'sqlserver') {
		...
	} else {
		throw 'Unsupported DB type: ' + dbType;
	}
}
